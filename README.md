# Gym Management App

An Android application for managing gyms and martial arts academies, covering student enrollment, class modalities, payments, and belt graduations.

---

## Architecture

The project follows the **MVVM (Model-View-ViewModel)** pattern with a clear three-layer separation:

```
app/
└── src/main/java/gym/management/
    ├── data/           # Data layer (repository implementations, Firebase)
    ├── domain/         # Domain layer (models, repository interfaces)
    ├── presentation/   # Presentation layer (ViewModels, Screens, Fragments)
    └── ui/theme/       # Theme, colors and typography
```

### Layers

**`domain/`**
- Data models: `Student`, `Modality`, `Payment`, `Graduation`, `User`
- Repository interfaces: `StudentRepository`, `ModalityRepository`, etc.
- Has no dependency on any other layer

**`data/`**
- Repository implementations backed by Firebase Firestore and Firebase Auth
- `FirestoreCollections` — constants for Firestore collection names
- Uses `callbackFlow` to expose real-time data streams as Kotlin `Flow`

**`presentation/`**
- Each feature module contains: `Fragment` + `Screen` (Compose) + `ViewModel` + `UiState`
- Fragments host Compose UI via `ComposeView`
- Navigation is handled by the **Navigation Component** (fragment nav graph)
- `UiState` is a `sealed class` representing: `Loading`, `Success`, `Error`

### Data Flow

```
Firestore → RepositoryImpl → ViewModel (StateFlow) → Screen (Compose) → UI
```

## Features

| Screen | Description |
|---|---|
| Login / Sign Up | User authentication via Firebase Auth |
| Main Menu | Entry point for all app sections |
| Students | List, register, and view detailed student profiles |
| Modalities | List, register, edit modalities and view enrolled students |
| Payments | Track monthly payments per student and modality |
| Graduations | Record belt promotions and grades per student |

## Data Models

### `Student`
| Field | Type | Description |
|---|---|---|
| `id` | String | Unique ID (generated by Firestore) |
| `name` | String | Student's full name |
| `phone` | String | Phone number |
| `address` | String | Home address |
| `birthDate` | String | Date of birth |
| `emergencyContactName` | String | Emergency contact name |
| `emergencyContact` | String | Emergency contact phone |
| `paymentDay` | Int | Monthly due day |
| `modalityIds` | List\<String\> | IDs of enrolled modalities |
| `active` | Boolean | Whether the student is active |
| `registrationDate` | Long | Enrollment date (timestamp) |

### `Modality`
| Field | Type | Description |
|---|---|---|
| `id` | String | Unique ID |
| `name` | String | Modality name |
| `schedules` | List\<String\> | Class times in HH:MM format |
| `frequency` | String | Days of the week (e.g. "Monday, Wednesday, Friday") |
| `price` | Double | Monthly fee |
| `active` | Boolean | Whether the modality is active |

### `Payment`
| Field | Type | Description |
|---|---|---|
| `studentId` | String | Reference to the student |
| `modalityId` | String | Reference to the modality |
| `amount` | Double | Amount paid |
| `month` / `year` | Int | Payment period |
| `isPaid` | Boolean | Payment status |
| `paidAt` | Long? | Timestamp of when payment was made |

### `Graduation`
| Field | Type | Description |
|---|---|---|
| `studentId` | String | Reference to the student |
| `modalityId` | String | Reference to the modality |
| `belt` | String | Belt awarded |
| `generalGrade` | String | Overall grade |
| `observation` | String | Additional notes |
| `date` | Long | Graduation date (timestamp) |

## Build Variants

The project uses a **`brand`** flavor dimension combined with two build types, producing four variants:

| Variant | Flavor | Build Type |
|---|---|---|
| `kiolJongDebug` | `kiolJong` | debug |
| `kiolJongRelease` | `kiolJong` | release |
| `gymManagementDebug` | `gymManagement` | debug |
| `gymManagementRelease` | `gymManagement` | release |

### Product Flavors

**`kiolJong`** — brand for Grupo Kiol Jong
- App name: `Grupo Kiol Jong`
- Logo: custom PNG asset
- Color palette: Gold (dark mode) / Red (light mode)

**`gymManagement`** — generic gym management brand
- App name: `Gym Management`
- Logo: custom vector drawable (`gym.xml`)
- Color palette: Pink/Magenta launcher background (`#CF0472`)
- Feature flag: `use_gym_logo = true`

### Build Types

**`debug`** — default development build, no signing configuration required.

**`release`** — production build signed via `keystore.properties` (see Setup). Proguard rules are applied but minification is currently disabled (`isMinifyEnabled = false`).

## Requirements

- **Language:** Kotlin
- **Build system:** Gradle with Kotlin DSL (`.gradle.kts`)

## Setup

1. Clone the repository
2. Create a project in the [Firebase Console](https://console.firebase.google.com) with **Authentication** (email/password) and **Firestore** enabled
3. Download the `google-services.json` file and place it inside `app/`
4. Open the project in Android Studio and run
